Lesson 2 Reflections

Question 1: What happens when you initialize a repository? Why do you need to do it?

Answer 2: Initializing a repository prepares it to begin tracking changes and commits. I need it so that I can keep track of my changes and updates.

Question 2: How is the staging area different from the working directory and the repository? What value do you think it offers?

Answer 2: The staging area is the area between the working directory and the repository it provides a way for you to group files together in one group before adding them to the repository.

Question 3: How can you use the staging area to make sure you have one commit per logical change?

Answer 3: By using the git diff to check the difference between the working directory and the staging area before commiting the files to the repository.

Question 4: What are some situations when branches would be helpful in keeping your history organized? How would branches help?

Answer 4: When I want to have separate files for separate languages or for separate audiences. Branches will help be create separate files for each language or audience.
